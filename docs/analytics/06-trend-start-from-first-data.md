# 访问趋势起始日期裁剪

## 功能目的
修复后台行为分析中的访问趋势图在首个有效数据前展示大量空白日期的问题，让曲线从首个有 PV 或 UV 的日期开始。

## 使用方式/入口
- 后台入口：行为分析页“访问趋势”图表。
- 接口入口：GET /activity/stats/trends?days=30。

当前行为：
- 后端仍然按近 N 天补齐缺失日期，保证趋势连续。
- 若补齐结果前段全部为 0，则会裁掉这些前导零值日期。
- 若整个时间窗口都没有数据，则保持原始结果，避免接口返回异常结构。

## 关键约束与边界
- 仅裁剪前导连续 0，不会移除中间的 0 值日期，趋势断点仍按真实时间线保留。
- 裁剪条件为 pv > 0 或 uv > 0 任一成立，防止只看 PV 或只看 UV 导致起点偏移。
- 该逻辑在服务端生效，因此行为分析页和其他复用趋势接口的页面会同步受益。
