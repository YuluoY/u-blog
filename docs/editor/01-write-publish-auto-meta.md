## 功能目的
在撰写页每次点击发布/更新时，自动从正文提取标题、详情（desc）、封面，减少手动维护元信息。
若用户已手动填写标题、简介或替换封面，则优先保留用户输入。
当正文无图片时，由后端自动生成默认封面图，保证封面字段始终可用。

## 使用方式/入口
- 前端入口：撰写页发布抽屉提交（`WriteSaveForm` 的 `handleSubmit`）
- 后端入口：`/rest/article/add` 与 `/rest/article/update`

发布/更新时行为：
1. 标题：优先使用用户填写值；未填写时再回退正文首个 Markdown 标题；仍无标题则使用正文摘要兜底。
2. 详情：优先使用用户填写值；未填写时自动从正文提取纯文本摘要（去除代码块/链接/图片/HTML 等标记）。
3. 封面：优先使用用户上传/输入的封面；未填写时再使用正文首张图片；若无图片则后端生成并保存默认 PNG 封面，返回 `/uploads/cover-article-*.png`。

## 关键约束与边界
- 前端仅在字段缺失时自动补齐，用户已手动修改的标题/简介/封面不会被覆盖。
- 发布抽屉再次打开时，只会补齐仍为空的标题/简介，不再覆盖用户已填写内容。
- 后端同样遵循“用户输入优先，自动提取兜底”的规则，避免更新时把手动封面替换回正文首图。
- 后端为最终兜底，即使前端未传封面，也会自动补齐默认封面。
- 默认封面为后端落盘 PNG 文件，风格统一且按标题自动变化配色。
- 默认封面采用固定布局模板（顶部 NOTES + 中心标题 + 底部站点名），与目标视觉一致。
- 默认封面标题会自动清理尾部时间戳，背景渐变为多风格概率随机。
- 默认封面标题在中间区域按固定边距自动换行展示，并使用较小字号。
