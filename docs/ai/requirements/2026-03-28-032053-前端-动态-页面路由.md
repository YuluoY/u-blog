## 原始需求
<!-- INITIAL_REQUEST_START -->
用户希望添加一个任何用户都可以访问的前端路由，名为「动态」，用于展示某种动态内容。
<!-- INITIAL_REQUEST_END -->

## 当前理解
<!-- CURRENT_UNDERSTANDING_START -->
「动态」是博主专属发布的短内容流，所有访客可浏览。博主通过前端动态页或 Admin 后台发布含文本（支持 Markdown）+ 可选图片的动态，附带可选的心情、标签、天气字段。展示采用卡片 + 时间线融合风格，带左右侧边栏，无限滚动加载。图片支持九宫格、合成长图、横向滚动 3 种展示模式。支持点赞和评论（复用现有评论系统）。支持公开/私密可见性。后端模型名为 Moment。
<!-- CURRENT_UNDERSTANDING_END -->

## 已确认事实
<!-- CONFIRMED_FACTS_START -->
- 发布权限：仅博主（站点所有者）可发布
- 浏览权限：所有用户（含游客）可浏览
- 内容类型：支持文本 + 可选图片，同时支持 Markdown 格式排版
- 交互能力：支持点赞 + 评论
- 发布入口：前端动态页内可发布（博主可见），Admin 后台可发布 + 管理
- 页面风格：卡片 + 时间线融合（单列居中卡片，左侧时间指示器，底部操作栏）
- 评论能力：复用现有博客文章评论系统
- 加载方式：无限滚动
- 图片展示：支持 3 种模式——九宫格、合成长图、横向滚动（发布时由博主选择）
- 导航位置：首页后面（index=2），其他路由 index 顺移
- 侧边栏：带左右侧边栏，沿用首页风格
- 附加字段：心情（可选）、标签（可选）、天气（可选，支持手动选择或 IP 自动获取）
- 内容限制：不限制文字长度和图片数量
- 可见性：支持公开/私密（仅博主可见）
- 模型命名：Moment（后端实体、API 路由、表名均以此为准）
- 评论展示：卡片上显示评论数，点击展开评论区（列表内展开，不跳转）
- 置顶：支持置顶，置顶动态始终显示在列表最顶部
- 心情选择：预定义常用 emoji 列表 + 完整 emoji 选择器

## 最终收敛总结
<!-- FINAL_SUMMARY_START -->
### 目标
为博客前端新增「动态」（Moments）功能模块，允许博主发布短内容动态，所有用户可浏览、点赞、评论。

### 目标用户
- 博主：发布、管理、置顶动态
- 所有访客（含游客）：浏览、点赞、评论

### 范围
1. **后端**：Moment 实体 + REST CRUD + 点赞接口 + 天气自动获取
2. **前端**：`/moments` 路由 + MomentsView 页面 + 发布框 + 卡片时间线 + 图片展示 + 评论展开
3. **Admin**：Moment CRUD 管理 + 置顶/可见性管理
4. **共享包**：Moment schema 定义

### 非范围
- 动态转发/分享功能
- 动态搜索/全文检索
- @提及其他用户
- 视频内容支持

### 关键约束
- 复用现有评论系统，不独立实现
- 图片展示模式由发布者选择（九宫格 / 合成长图 / 横向滚动）
- 心情选择器提供预定义 emoji + 完整 emoji picker
- 天气字段支持手动选择或 IP 自动获取
- 导航栏 index=2（首页后面），其他路由 index 顺移
- 页面带左右侧边栏，沿用首页风格

### 验收标准
- 游客可无限滚动浏览公开动态
- 博主可在前端页面和 Admin 后台发布、编辑、删除、置顶动态
- 私密动态仅博主可见
- 图片展示支持 3 种模式切换
- 评论区在卡片内展开/收起，显示评论计数
- 点赞可正常 toggle
<!-- FINAL_SUMMARY_END -->
